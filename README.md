# html-preview

> **Single × Stand Alone** — 1ファイルに全部詰め込む、徹頭徹尾の孤独主義。

ブラウザさえあれば動く、インストール不要のHTMLツール集。
`npm install` なんて知らない。`package.json` は幻。世界はHTMLファイル1枚で足りる。

## AI が生成したHTMLの、最終的な置き場所

AIエージェントが急成長している。Cursor がある。Claude Code がある。GitHub Copilot がある。
コードを書かずとも、エディタがコードを書いて、実行して、修正までしてくれる時代になった。

**あなたには関係ない話だが。**

あなたにインストールできるのは、会社が許可したソフトウェアだけだ。
エージェント系ツールの申請は、承認まで2週間かかる。そもそも、VSCode が重い。

しかし、ブラウザは使える。ChatGPT も、Claude も、ブラウザで動く。
あなたは今日もブラウザのタブに向かって「このHTMLを作って」とお願いした。
AIは答えた。きれいなコードブロックで。

**問題は、そのあとだ。**

生成されたHTMLを、どこで確認するのか。

ローカルサーバー？ないです。
VS Code + Live Server？重いし申請が必要です。
`.html` ファイルをダブルクリックしてブラウザで開く？
— それで確認できるものもあるが、CSS変数や iframe を使ったコードは崩れることがある。

そこでこのツールを開く。貼り付ける。確認する。
**3ステップ。追加インストール、ゼロ。**

**Vibe Coding の最終受取所として:**

```
# HTMLページを作りたい場合
1. ChatGPT / Claude に「こういうHTMLを作って」と頼む
2. 生成されたコードをコピーする
3. designer.html を開いて貼り付ける → リアルタイムプレビューで確認
4. 気に入らなければデザインモードで直接いじる
5. Ctrl+S でダウンロード

# スライドを作りたい場合
1. ChatGPT / Claude に「スライド用のHTMLを作って」と頼む
2. 生成されたコードをコピーする
3. slides.html を開いて貼り付ける → スライド単位でプレビュー確認
4. F5 でプレゼンモードに入って、そのまま発表する
```

あなたは「お話しかできない」と思っているかもしれない。
でも、AIに作らせて、このツールで確認して、調整して、保存する。それは立派なワークフローだ。

Claude Code を使っている人たちと同じゴールに、別のルートで辿り着いている。
回り道かもしれないが、道は道だ。

## ツール一覧

### `designer.html` — Visual HTML Designer

HTMLをリアルタイムで書いて、即座にプレビューを見ながら直接編集できるエディタ。

**主な機能**

- CodeMirror 搭載のシンタックスハイライトエディタ
- リアルタイムプレビュー（iframe）
- **デザインモード** `Ctrl+D` — 要素をクリックして色・フォント・余白をGUI操作
  - 色変更（背景色・文字色・ボーダー色）
  - タイポグラフィ（フォントサイズ・太さ・行間・揃え）
  - スペーシング（padding・margin）
  - ボックス（幅・高さ・ボーダー・角丸・影）
  - 要素の移動・複製・削除・スタイルコピペ
- レイアウト切り替え（左右分割 / 上下分割 / プレビューのみ）
- ダーク/ライトテーマ切り替え
- HTMLファイルを開く / ダウンロード保存 `Ctrl+S`
- Undo/Redo、コードフォーマット、コードの折りたたみ、検索

**使い方**

`designer.html` をブラウザで開く。以上。

---

### `slides.html` — Slide Presentation Editor

HTMLでスライドを作って、ブラウザ上でプレゼンするためのエディタ。

**主な機能**

- HTMLまたはMarkdownでスライドコンテンツを記述
- スライド単位での管理（追加・削除・ナビゲーション）
- アスペクト比切り替え（16:9 / 4:3）
- **プレゼンモード** `F5` — フルスクリーン発表ビュー
- 印刷 `Ctrl+P`（PDF出力にも使える）
- **デザインモード** `Ctrl+D` — designer.html と同様のGUI編集
- スライドパネル（サムネイル一覧）
- ダーク/ライトテーマ切り替え
- ファイルを開く / ダウンロード保存 `Ctrl+S`

**使い方**

`slides.html` をブラウザで開く。以上。

## 設計思想

```
制約: Single × Stand Alone
```

- **1ファイル = 1ツール** : HTML・CSS・JSがすべて1枚のファイルに収まっている
- **インストール不要** : ブラウザで開けば動く
- **依存ゼロ（ローカル）** : CDNのライブラリは使うが、手元には何も置かない
- **オフライン不可** : CDNに繋がらなければ動かない。その潔さも美学

これは「制約」ではなく「仕様」。
1ファイルに収まらないものは、このリポジトリには存在しない。
